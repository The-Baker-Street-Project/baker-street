# Wave 5 & 6 Plan

Date: 2026-03-07

## Wave 5: Local Models & Model Stack

**Goal:** Any user can run Baker Street with local models (Ollama/vLLM/SGLang/vllm-mlx) as brain and/or worker, with independent model selection per role. The model layer becomes comprehensive with provider registry and embeddings abstraction.

### Issues

| Issue | Title | Size | Priority | Dependency |
|-------|-------|------|----------|------------|
| BAK-53 | Enable local models as Brain agent (tool calling for OpenAI-compatible adapter) | Large | High | — |
| BAK-28 | Deferred multi-model features: Google adapter, provider registry, per-request routing | Medium | Normal | BAK-53 |
| BAK-32 | Model-as-a-Service support (Voyage embeddings, provider abstraction) | Medium | Normal | BAK-53 |

### Execution Order

1. **BAK-53** lands first — it unblocks everything:
   - Fix `convertToOpenAIMessages()` in `packages/shared/src/model-router.ts` to use `convertToOpenAIMessagesWithTools()` (the helper already exists, just needs wiring)
   - Add `WORKER_MODEL` env var to `packages/shared/src/model-config.ts` for independent brain/worker model selection
   - Update installer model picker with local model suggestions
   - Test with Qwen 3.5 on Ollama (Sherlock) and vLLM

2. **BAK-28** and **BAK-32** run in parallel after BAK-53:
   - BAK-28: Google Vertex adapter, provider registry pattern, per-request model routing
   - BAK-32: Abstract embedding providers (currently hardcoded to Voyage), support Ollama embeddings as local alternative

### Key Technical Notes

- Root cause blocker: `model-router.ts` line ~554 — OpenAI-compatible adapter calls `convertToOpenAIMessages()` which strips tool blocks. The OpenAI native adapter (line ~448) already uses `convertToOpenAIMessagesWithTools()` which preserves them. One-line fix plus tests.
- `ModelRoles` already supports `worker?: string` — just needs env var wiring and installer support.
- All local inference servers (Ollama, vLLM, SGLang, llama.cpp) expose OpenAI-compatible APIs, so fixing one adapter covers all of them.
- vLLM continuous batching matters because brain fires multiple LLM roles simultaneously (agent, observer, reflector, workers).

### What's Already Done

- BAK-29 (Ollama auto-discovery) — merged
- BAK-30 (UI model selector) — merged
- BAK-18 (multi-model providers) — merged
- ModelRouter with provider adapters — working for Anthropic, OpenRouter, OpenAI native
